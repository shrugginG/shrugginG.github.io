在这个示例中我们为函数open()提供了两个参数，第一个是指定要写入的文件名，第二个‘w’是要指示python我们要以写入模式打开这个文件。打开文件时我们可以指定读取模式(‘r)，写入模式(‘w’)，附加模式(‘a’)或让你能够读取和写入的模式(‘r+’)。如果我们省略第二个形参那么python将默认以读取模式打开这个文件。

python只能文本以字符串的形式写入文件，若想将数值写入文件注意要先将数值转换为字符串。

写入多行

注意write()并不会在我们每次写入时在每行的末尾添加换行符，因此我们如果需要换行时就需要自己手动添加换行符/n。

附加到文件

如果我们要向文件中添加内容而不是覆盖原有内容就需要用到附加模式打开文件。在使用附加模式打开文件时我们新写入的内容就会自动添加到文件的末尾。

异常

python使用成为异常的特殊对象来管理程序执行期间发生的错误。每当发生让python不知所措的错误是它都会创建一个异常对象。如果我们编写了处理该异常的代码那么程序将继续运行，否则程序将停止运行并显示一个trackback其中包含有关异常的报告。

python中对异常的处理是使用try-except代码块处理的。try-except代码块让python执行指定的操作，同时告诉python遇到异常时该怎么办。

使用异常避免崩溃

发生错误时如果程序还没有运行完毕那么妥善处理错误就显得尤为重要。这种情况经常发出现在要求用户提供输入的程序之中，如果用户并没有按照要求输入有效的内容我们就需要提醒用户正确输入。

else代码块

我们可以将可能引发错误的代码放入try-except代码块中这样就可以提高我们这个程序抵御错误的能力。

try-except-else代码块的工作原理大致如下：python尝试执行try代码块中的代码，只有可能引发异常的代码才需要放在try语句中。有时候有一些仅在try代码块成功执行时才需要运行的代码。这些代码应该放在else代码块中。except代码块告诉python，如果它尝试运行try代码中的代码而出现错误时并引发了指定的异常该如何处理。

处理FileNotFoundError异常

使用文件时一种常见的异常是我们找不到文件，我们要查找的文件可能在其他地方、文件名可能不正确或者这个文件根本就不存在。对于这种情形都可使用try-except代码块以直观的方式进行处理。

保存数据的一种简单的方式就是使用模块json来存储数据。

Json(JavaScriptObjectNotation)格式最初是为JavaScript开发的，但随后成为了一种常见格式，被包括python在内的多种语言所采用。
